Google Search in Top Navigation Output

As an alternative to MadCap Search, you can now use Google Search for your HTML5 Top Navigation output.

Using Google Search involves a combination of steps on your Google Internet account, in addition to a few steps in Flare. Not all of the Google account steps outlined below are required for integrating search with Flare, but they are recommended. Also, the exact steps that you follow may be somewhat different if you are using a free Google account as opposed to a paid account.

Google Search Versus MadCap Search

There are benefits to both the Google and MadCap Search methods.

 

Google Search

MadCap Search

Quality

Results are typically better, depending on the search performed.

Results might not be as robust, depending on the search performed.

Public Versus Local Output

Output must be available publicly so that you can use Google. Output cannot be behind a firewall.

Output can be searched whether it is available publicly or locally.

Cost

You can set up a free Google account, but the features are limited. Alternatively, you can use a paid Google account, which can lessen the burden but you will be paying for the extra features. This could become significant if you have a great many pages in your outputs.

There is no extra cost in using MadCap Search.

Search Speed

Search results might lag at times, depending on the size of the output and whether Google is finished with the indexing (i.e., making your output available for searches).

Google will index your pages automatically over time, but it can take awhile, sometimes weeks. You can use some of the steps below to help speed up this process. This is quicker and easier with a paid Google account than it is with a free account.

In addition, there are a number of different factors that can determine how fast your site is indexed, including:

  • Whether your output has actual content
  • Whether your output has a sitemap
  • The structure of your output
  • How content is linked (regular links are best to make it easier for Googlebots to crawl)
  • Popularity of the site (the number of other sites that link to your output)

Search results might be faster because the output is immediately indexed once it is generated. You do not need to spend any extra time indexing your pages.

Mobile Devices

Search results always come from a server, so you do not need to be concerned about the ability of the mobile device to handle the information.

Search results are “chunked” and then collapsed so that they can be displayed when a search is performed. This is usually okay on PCs, but if you try to perform a search on a mobile device, the results might be slow or even crash because the device cannot handle the number of search results.
Google Account—Create Search Engine
  1. Go to cse.google.com and log in with your Google (i.e., gmail) password.
  2. Click Add to create a custom search engine.

  3. In the Sites to search field, enter the URL for the site where your output will be published. A name for the search engine is automatically added toward the bottom of the page, based on your URL. You can edit the name to something more simple if you want.

  4. Click Create.
  5. On the next page, click Control Panel to retrieve your search engine ID. (You can also use one of the other methods—Get code or Public URL—if you prefer.) Then click Search engine ID.

  6. Copy the ID shown in the popup.

  7. Continue with the following Flare steps to enter the Google search key and generate a sitemap.
 Flare Project—Enter Google Search Key and Generate Sitemap
  1. Open an HTML5 target.
  2. Select the Search tab.
  3. From the Search Engine field, select Google Search.

  4. In the Google Search Context Key field, paste the ID that you copied from cse.google.com.

  5. To assist with indexing, we also recommend that you create a sitemap. So Click Generate Sitemap and enter the same URL that you provided on your Google account page (make sure you include http:.// before the path).

  7. Generate the target and upload the output files to your URL.
  8. Continue with the following steps to verify your URL.

 Google Account—Exclude TOC from Indexing

By default, Google will index each word it finds on your pages, but there may be some words that you do not want to be indexed. This likely includes your TOC. After all, the words in the TOC will be repeated on most, if not all, of your pages. So it wouldn’t be useful for those words to be included in searches.

  1. Go back to the first browser tab for your Google account and click the Advanced tab.

  2. Click CSE context to expand that section.
  3. Click Download (XML). The cse.xml file is added to the Downloads folder on your computer.

  4. Edit the downloaded context file cse.xml to add the following attribute: enable_nocontent_tag=“true”.

    To do this, you can open the file in a text editor such as Notepad++. Then near the top of the file, change <CustomSearchEngine id= “id” creator=“[id]”> to <CustomSearchEngine id= “[id]” creator= “id” enable_nocontent_tag=“true”>. There may be other attributes that are already in the tag as well.

 Google Account—Customize Appearance of Search Results

By default, search results are overlayed on top of your output. You can change this look if you want.

  1. Click Look and feel.

  2. On the Layout tab, select an option. To make the search results appear the way that they normally do in Top Navigation output, select Results Only.

  3. Click Save.

Favicons (short for “favorite icons”) are now supported in for output. A favicon is the small image that you sometimes see in the tab, address bar, bookmark, or other areas of a website.

 Fixed header in Top Navigation Output

When generating Top Navigation output, you can fix the header portion of the topic so that it stays in place when users scroll down to see more content.

  1. Open an skin.
  2. Select the Setup tab.
  3. In the Fixed Header field, select one of the options:
    • None The header will not be fixed in the output.
    • Web The header will be fixed on screens that are larger than tablet and mobile.
    • Tablet & Mobile The header will be fixed on tablet and mobile screens.
    • Mobile The header will be fixed on mobile screens.
    • All The header will be fixed on all screens—web, tablet, and mobile.

 Tree Format in Top Navigation Off-Canvas Menus

In previous versions, the off-canvas menu for Top Navigation output (i.e., the menu that opens when you click the “hamburger” icon) always displayed in a drilldown format. With this format, any item containing sub-items has a double-arrow next to it. When you click on such an item, the entire menu changes, showing only that item and its sub-items.

Starting with this version, you have the option of using a tree format instead. With this format, any item containing sub-items has a down arrow next to it. when you click on that item, the sub-items are shown, but the rest of the menu remains in view as well.

Topic Headings in Top Navigation Side Menus

When creating a side menu in Top Navigation output, only topics were shown in previous versions. Starting with this version, you also have the option of showing topic headings in the side menu instead. In other words, any subheadings (e.g., h2, h3, h4) that you’ve included in a topic will be shown in the side menu instead of other topics in the output.

How to Use Topic Headings in a Topic Navigation Side Menu

  1. Insert a Menu proxy into a content file (e.g., topic, master page) or edit an existing one (right-click on the proxy and select Edit Menu Proxy). See Creating a Topic Menu.

  2. In the Menu Proxy dialog, click in the first drop-down field and select (Topic Headings).
